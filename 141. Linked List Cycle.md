Givenô `head`, the head of a linked list, determine if the linked list has a cycle in it.

There is a cycle in a linked list if there is some node in the list that can be reached again by continuously following theô `next`ô pointer. Internally,ô `pos`ô is used to denote the index of the node thatô tail'sô `next`ô pointer is connected to.ô **Note thatô `pos`ô is not passed as a parameter**.

Returnô `true`ô _if there is a cycle in the linked list_. Otherwise, returnô `false`.

**Example 1:**

![](https://assets.leetcode.com/uploads/2018/12/07/circularlinkedlist.png)

**Input:** head = [3,2,0,-4], pos = 1
**Output:** true
**Explanation:** There is a cycle in the linked list, where the tail connects to the 1st node (0-indexed).

**Example 2:**

![](https://assets.leetcode.com/uploads/2018/12/07/circularlinkedlist_test2.png)

**Input:** head = [1,2], pos = 0
**Output:** true
**Explanation:** There is a cycle in the linked list, where the tail connects to the 0th node.

**Example 3:**

![](https://assets.leetcode.com/uploads/2018/12/07/circularlinkedlist_test3.png)

**Input:** head = [1], pos = -1
**Output:** false
**Explanation:** There is no cycle in the linked list.

**Constraints:**

- The number of the nodes in the list is in the rangeô `[0, 104]`.
- `-105 <= Node.val <= 105`
- `pos`ô isô `-1`ô or aô **valid index**ô in the linked-list.

**Follow up:**ô Can you solve it usingô `O(1)`ô (i.e. constant) memory?



# Brute Force

```python
# Definition for singly-linked list.

# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        visited = set()
        current = head
        while current is not None:
            if current in visited:
                return True
            else:
                visited.add(current)
            current = current.next
        return False
```



# Approach

This is based on Python. Other might be different a bit.

1. Initializeô `fast`ô andô `slow`ô pointers to the head of the linked list.
    
    - `fast`ô andô `slow`ô are initially pointing to the head of the linked list, which is the starting point.
2. Enter a loop while bothô `fast`ô andô `fast.next`ô are not null.
    
    - This loop will continue until eitherô `fast`ô orô `fast.next`ô becomes null, indicating the end of the linked list or if there is no cycle.
3. Move theô `fast`ô pointer two steps ahead (`fast = fast.next.next`) and theô `slow`ô pointer one step ahead (`slow = slow.next`).
    
    - This step simulates the two-pointer traversal whereô `fast`ô moves twice as fast asô `slow`.
4. Check ifô `fast`ô is equal toô `slow`.
    
    - Ifô `fast`ô andô `slow`ô pointers meet at the same node, it means there is a cycle in the linked list, and we returnô `True`.
5. If the loop exits without finding a cycle, returnô `False`.
    
    - If the loop completes without finding a cycle, it means that theô `fast`ô pointer has reached the end of the list, and there is no cycle. In this case, we returnô `False`.

This algorithm uses two pointers,ô `fast`ô andô `slow`, to traverse the linked list. Theô `fast`ô pointer moves twice as fast as theô `slow`ô pointer. If there is a cycle, theô `fast`ô pointer will eventually catch up to theô `slow`ô pointer. If there is no cycle, theô `fast`ô pointer will reach the end of the list, and the function will returnô `False`. This algorithm is an efficient way to detect cycles in a singly-linked list.


```python
class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
    
        fast = head
        slow = head
        
        while fast and fast.next:
            fast = fast.next.next
            slow = slow.next
            
            if fast == slow:
                return True
    
        return False
```

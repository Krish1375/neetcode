Given two strings `text1` and `text2`, return _the length of their longest **common subsequence**._ If there is no **common subsequence**, return `0`.

A **subsequence** of a string is a new string generated from the original string with some characters (can be none) deleted without changing the relative order of the remaining characters.

- For example, `"ace"` is a subsequence of `"abcde"`.

A **common subsequence** of two strings is a subsequence that is common to both strings.

**Example 1:**

**Input:** text1 = "abcde", text2 = "ace" 
**Output:** 3  
**Explanation:** The longest common subsequence is "ace" and its length is 3.

**Example 2:**

**Input:** text1 = "abc", text2 = "abc"
**Output:** 3
**Explanation:** The longest common subsequence is "abc" and its length is 3.

**Example 3:**

**Input:** text1 = "abc", text2 = "def"
**Output:** 0
**Explanation:** There is no such common subsequence, so the result is 0.

**Constraints:**

- `1 <= text1.length, text2.length <= 1000`
- `text1` and `text2` consist of only lowercase English characters.

# Approach

```javascript
Input: text1 = "abcde", text2 = "ace"
Input: text1 = "abcd", text2 = "dbca"
```

Let’s start by looking at some strings. In the first case, we have strings that are very simple, so we find the subsequence `"ace"`. Both strings contain `"ace"`, so the length of the subsequence is `3`.

Now, let’s look at the second example: the first string is `"abcd"` and the second string is `"dbca"`. We find `"a"` in the second string, but it’s at the last index, so the length is `1`. Next, we look at `"bc"`, and the second string also has `"bc"`, so the length should be `2`. Then, we find `"d"` in the first string, and the second string has `"d"` at the first index, so the length is `1`.

From this second example, **we can see that the last character is not always part of the longest subsequence.** That’s why we need to check the lengths of the subsequences and keep track of them.

To do this, we use dynamic programming.

## How can we implement that?

Let’s create a DP list and initialize it with zeros, with a length equal to the length of the first string.

```javascript
Input: text1 = "abcd", text2 = "dbca"

      a,b,c,d (= text1)
dp = [0,0,0,0]
longest = 0
```

Let’s take an example where the length is `4`, so we have `[0,0,0,0]`. Next, we initialize a variable `longest` to zero and iterate through each character in the second string.

So, let’s begin. The first character is `"d"` in `text2`. We try to find `"d"` in `text1`, and there is a `"d"` at index `3`. Therefore, we update the DP list at index `3` with the value `1`. The DP list is now used to keep track of the longest common subsequence at each position. So, the current longest subsequence is `1`.

```
      a,b,c,d (= text1)
dp = [0,0,0,1]
longest = 1
```

We move to the next character, which is `"b"`. We find `"b"` at index `1` in `text1`, so we update the DP list at index `1` with the value `1`. The longest subsequence remains `1`.

```
      a,b,c,d (= text1)
dp = [0,1,0,1]
longest = 1
```

Then, we find `"c"`, and we find `"c"` at index `2` in `text1`, so we update index `2` with `1`.

```
      a,b,c,d (= text1)
dp = [0,1,1,1]
longest = 1
```

But here’s a problem.

```cpp
          -- → Longest Common Subsequence(= "bc")
text1 = "abcd"
         -  -

          --
text2 = "dbca"
         -  -
```

At the position of `"c"`, the longest common subsequence should be `2` because the `text2` has `"bc"` and `text1` also has `"bc"`. However, we only have `1` in the DP list at index `2`, which is incorrect.

## How do we handle this problem?

Here’s the strategy: since we have two strings, we will use a nested loop. In the outer loop, we iterate through the second string, and each time we start the outer loop, we initialize a variable `current_length` to zero. This variable keeps track of the current longest common subsequence.

In the inner loop, we iterate through the DP list instead of `text1`. By doing this, **we can get the index number and each value in the DP list.** When we iterate through the DP list with `"c"`, the DP list should be `[0, 1, 0, 1]`. We start from the beginning, and each time we compare the current data in DP list with the `current_length`. If `current_length` is less than the current value, we update `current_length` with the current value in the DP list.

So let’s start from `c` in `text2`. First, we find `0` at index `0`(= `a`) in DP list, and since `current_length` is `0`, we don’t update it. The characters are different between two strings(= `c` and `a`), so we don’t do anything.

```sql
Input: text1 = "abcd", text2 = "dbca"
                                  ↑ (= current character)
      a,b,c,d (= text1)
dp = [0,1,0,1]
      ↑
current_length = 0
longest = 1
```

We move to the next character in DP list, which is `"b"`. We compare the current data in DP list with `current_length`, and since the current data in DP list is `1`, we update `current_length` to `1`.

```sql
Input: text1 = "abcd", text2 = "dbca"
                                  ↑ (= current character)
      a,b,c,d (= text1)
dp = [0,1,0,1]
        ↑
current_length = 1
longest = 1
```

**This means that at the position of `"b"`, we have at least one character as the longest common subsequence.**

We move on in DP list, and we find `"c"`. We compare the current data in DP list with `current_length`, and since `current_length` is longer than the number at index `2` in DP list, we don’t update `current_length`.

```sql
Input: text1 = "abcd", text2 = "dbca"
                                  ↑ (= current character)
      a,b,c,d (= text1)
dp = [0,1,0,1]
          ↑
current_length = 1
longest = 1
```

But now we find the same character `"c"`, at index `2`, so we update the DP list at index `2` with `current_length + 1`(= `"bc"`), which is `2`. Now, the longest subsequence at `"c"` is `2`, which is correct.

```python
          -- → At "c", LCS is 2
text2 = "dbca"
         -  -

      a,b,c,d (= text1)
dp = [0,1,2,1]
            ↑
current_length = 1
longest = 2
```

We move on to `"d"`. Since `1` is equal to `current_length` and the characters are different(`c` and `d`), so we don’t do anything.

```
      a,b,c,d (= text1)
dp = [0,1,2,1]
current_length = 1
longest = 2
```

In the end, we get `[0, 1, 2, 1]` and the longest subsequence length is `2`.

Next, we find `"a"`. We do the same thing and iterate through the DP list from the beginning to the end. Before we start the outer loop, we reset `current_length` to zero.

```sql
Input: text1 = "abcd", text2 = "dbca"
                                   ↑ (= current character)
      a,b,c,d (= text1)
dp = [0,1,2,1]
      ↑
current_length = 0
longest = 2
```

When we find the same character `"a"` at index `0`, we update index `0` in the DP list with `0 + 1`, which is `1`.

```sql
Input: text1 = "abcd", text2 = "dbca"
                                   ↑ (= current character)
      a,b,c,d (= text1)
dp = [1,1,2,1]
　　　　　　　　　　　　↑
current_length = 0
longest = 2
```

Moving on, we compare the current data(`"b"`) with `current_length`, and since `current_length` is less than `"b"` at index `1`, update `current_length` with the number at index `1`. The characters are different(= `a` and `b`), so we don’t do anything further. We continue until the end. Let me skip rest of explanation.

```sql
Input: text1 = "abcd", text2 = "dbca"
                                   ↑ (= current character)
      a,b,c,d (= text1)
dp = [1,1,2,1]
　　　　　　　　　　　　  ↑
current_length = 1
longest = 2
```

Finally, the `longest` variable holds the length of the longest common subsequence, so we should return that value.

One important point is that when we update `longest`, we should use the formula `max(longest, current_length + 1)`.

Let me explain why. When we iterate through the DP list with `"a"`, the DP list should look like `[0, 1, 2, 1]` before we start. After updating index `0` with `0 + 1`, it becomes `1`. If we then update `longest`, we would incorrectly set it to `1` instead of `2`. This is because before iterating through the DP list with "a", we already found a subsequence with a length of 2, "bc". That’s why we need to compare `longest` with `current_length + 1` every time. This way, we correctly keep `longest` as 2.

So, that’s the important point to remember. The `longest` variable is updated using the formula `max(longest, current_length + 1)` to ensure we don’t lose track of the longest subsequence.

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        dp = [0] * len(text1)
        longest = 0

        for c in text2:
            cur_length = 0
            for i, val in enumerate(dp):
                if cur_length < val:
                    cur_length = val
                elif c == text1[i]:
                    dp[i] = cur_length + 1
                    longest = max(longest, cur_length + 1)
        
        return longest
```

#### Step 1: DP Array Initialization

A list `dp` is initialized with zeros, having a length equal to `text1`. This list will keep track of the longest common subsequence length at each position in `text1`.

#### Step 2: Initialize Longest Variable

```python
longest = 0
```

A variable `longest` is initialized to 0. This variable will store the maximum length of the common subsequence found so far.

#### Step 3: Iterate Through Text2

```python
for c in text2:
```

This `for` loop iterates over each character `c` in `text2`. The loop will compare each character in `text2` with characters in `text1` to find the common subsequence.

#### Step 4: Initialize Current Length

```python
cur_length = 0
```

At the start of each iteration over `text2`, a variable `cur_length` is set to 0. This variable tracks the longest subsequence length found in the current iteration of `text2`.

#### Step 5: Iterate Through DP List

```python
for i, val in enumerate(dp):
```

This nested `for` loop iterates over the `dp` list. The index `i` and the value `val` at each index are used to compare the current character `c` from `text2` with characters in `text1`.

#### Step 6: Update Current Length

```python
if cur_length < val:
    cur_length = val
```

If `cur_length` is less than `val` (the current value in `dp`), update `cur_length` to `val`. This means that the current character `c` from `text2` matches a subsequence in `text1` up to this point with a length of `val`.

#### Step 7: Update DP Array and Longest Subsequence

```python
elif c == text1[i]:
    dp[i] = cur_length + 1
    longest = max(longest, cur_length + 1)
```

If the current character `c` from `text2` matches the character at index `i` in `text1`, update `dp[i]` to `cur_length + 1`, representing an extension of the current subsequence by one more character.

After updating `dp[i]`, update `longest` to the maximum of its current value and `cur_length + 1`. This ensures that `longest` always holds the maximum length of the subsequence found so far.

#### Step 8: Return the Result

```python
return longest
```

The method returns the value of `longest`, which is the length of the longest common subsequence between `text1` and `text2`.

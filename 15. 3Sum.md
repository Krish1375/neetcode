[15. 3Sum](https://leetcode.com/problems/3sum/)
[[-Pointers]][[- Medium Level]]
#pointer #medium 

Given an integer array nums, return all the triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, and `j != k`, and `nums[i] + nums[j] + nums[k] == 0`.

Notice that the solution set must not contain duplicate triplets.

**Example 1:**

**Input:** nums = [-1,0,1,2,-1,-4]
**Output:**  -1,-1,2,-1,0,1
**Explanation:** 
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0.
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0.
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0.
The distinct triplets are [-1,0,1] and [-1,-1,2].
Notice that the order of the output and the order of the triplets does not matter.

**Example 2:**

**Input:** nums = [0,1,1]
**Output:** []
**Explanation:** The only possible triplet does not sum up to 0.

**Example 3:**

**Input:** nums = [0,0,0]
**Output:** [0,0,0]
**Explanation:** The only possible triplet sums up to 0.

**Constraints:**

- `3 <= nums.length <= 3000`
- `-105 <= nums[i] <= 105`



```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        ans = set()
        
        for i in range(len(nums)):
            if i > 0 and nums[i] == nums[i-1]:
                continue
            
            left = i + 1
            right = len(nums) - 1

            while left < right:
                total = nums[i] + nums[left] + nums[right]
                if total == 0:
                    ans.add((nums[i], nums[left], nums[right]))
                    left += 1
                    right -= 1
                    while left < right and nums[left] == nums[left - 1]:
                        left += 1
                    while left < right and nums[right] == nums[right + 1]:
                        right -= 1
                elif total < 0:
                    left += 1
                else:
                    right -= 1
                
        return ans
```

Okay, let's break down this Python code step by step.

The goal of this `threeSum` function is to find all **unique** combinations of three numbers in the input list `nums` that add up to zero.

**Core Idea:**

The strategy used is called the **Two Pointers** technique, applied after sorting the input array. Sorting is crucial because it allows us to efficiently search for the remaining two numbers once we've picked one.

**Step-by-Step Explanation:**

1. **`nums.sort()`:**
    
    - **Action:** The input list `nums` is sorted in ascending order.
    - **Why:** Sorting is essential for the two-pointer approach. It allows us to efficiently move pointers inward or outward based on whether the current sum is too small or too large. It also groups identical numbers together, making it easier to skip duplicates.
2. **`ans = set()`:**
    
    - **Action:** An empty set named `ans` is initialized.
    - **Why:** A set is used to store the resulting triplets. Sets automatically handle uniqueness; if we try to add the same triplet (even in a different order, although sorting prevents this specific case) multiple times, it will only be stored once. We store triplets as tuples `(nums[i], nums[left], nums[right])` because lists are mutable and cannot be added to sets, while tuples are immutable.
3. **`for i in range(len(nums))`:**
    
    - **Action:** This loop iterates through each number in the sorted `nums` list. The variable `i` represents the index of the _first_ number we are considering for a potential triplet.
    - **Why:** We need to consider each number as a potential first element of a zero-sum triplet.
4. **`if i > 0 and nums[i] == nums[i-1]: continue`:**
    
    - **Action:** This is a check to skip duplicate _first_ numbers. If the current number `nums[i]` is the same as the previous number `nums[i-1]`, we skip the rest of the loop for this `i` and move to the next `i`. The `i > 0` condition prevents checking `nums[-1]` when `i` is 0.
    - **Why:** Since the array is sorted, identical numbers are adjacent. If `nums[i]` is the same as `nums[i-1]`, any triplets found using `nums[i]` as the first element would be duplicates of triplets already found using `nums[i-1]`. Skipping avoids redundant work and duplicate results.
5. **`left = i + 1`** and **`right = len(nums) - 1`:**
    
    - **Action:** Two pointers, `left` and `right`, are initialized for _each_ iteration of the outer loop (for each `nums[i]`).
        - `left` starts at the index immediately _after_ `i`.
        - `right` starts at the very _last_ index of the list.
    - **Why:** We are looking for two other numbers (`nums[left]` and `nums[right]`) in the rest of the sorted array (from index `i+1` to the end) such that `nums[i] + nums[left] + nums[right] == 0`.
6. **`while left < right:`:**
    
    - **Action:** This inner loop continues as long as the `left` pointer is to the left of the `right` pointer.
    - **Why:** If `left` and `right` cross or meet, it means we have checked all possible pairs for the current `nums[i]`.
7. **`total = nums[i] + nums[left] + nums[right]`:**
    
    - **Action:** Calculate the sum of the three numbers currently pointed to by `i`, `left`, and `right`.
    - **Why:** To check if they sum up to the target (0).
8. **`if total == 0:`:**
    
    - **Action:**
        - A triplet summing to zero is found! Add the tuple `(nums[i], nums[left], nums[right])` to the `ans` set.
        - Increment `left` (`left += 1`).
        - Decrement `right` (`right -= 1`).
        - **Skip Duplicates (Inner):** Add two nested `while` loops to skip over any duplicate numbers at the _new_ `left` and `right` positions. This prevents adding the same combination multiple times if there are duplicate numbers in the middle or at the end (e.g., `[-2, 0, 0, 2, 2]` finding `(-2, 0, 2)` twice).
    - **Why:** We found a valid triplet. We add it to our results. We then move _both_ pointers inward because if we only moved one, we couldn't possibly get zero again with the _same_ `nums[i]` (since we'd be replacing one number from a zero-sum triplet with a different number). The inner duplicate skipping is crucial for uniqueness when `nums[left]` or `nums[right]` have duplicates.
9. **`elif total < 0:`:**
    
    - **Action:** Increment `left` (`left += 1`).
    - **Why:** The sum is too small. Since the array is sorted, moving the `left` pointer to the right will increase the sum (or keep it the same if the next element is a duplicate).
10. **`else: # total > 0`**
    
    - **Action:** Decrement `right` (`right -= 1`).
    - **Why:** The sum is too large. Since the array is sorted, moving the `right` pointer to the left will decrease the sum (or keep it the same if the next element is a duplicate).
11. **`return ans`:**
    
    - **Action:** After the outer loop finishes checking all possibilities, the function returns the `ans` set, which contains all unique triplets that sum to zero. Note: The type hint suggests `List[List[int]]`, but the code actually returns a `set` of `tuples`. Often, you might convert this set of tuples back to a list of lists before returning if strict type adherence is needed: `return [list(t) for t in ans]`.

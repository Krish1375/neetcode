Determine if a `9 x 9` Sudoku board is valid. Only the filled cells need to be validated **according to the following rules**:

1. Each row must contain the digits `1-9` without repetition.
2. Each column must contain the digits `1-9` without repetition.
3. Each of the nine `3 x 3` sub-boxes of the grid must contain the digits `1-9` without repetition.

**Note:**

- A Sudoku board (partially filled) could be valid but is not necessarily solvable.
- Only the filled cells need to be validated according to the mentioned rules.

**Example 1:**

![](https://upload.wikimedia.org/wikipedia/commons/thumb/f/ff/Sudoku-by-L2G-20050714.svg/250px-Sudoku-by-L2G-20050714.svg.png)

**Input:** board = 
[["5","3",".",".","7",".",".",".","."]
,["6",".",".","1","9","5",".",".","."]
,[".","9","8",".",".",".",".","6","."]
,["8",".",".",".","6",".",".",".","3"]
,["4",".",".","8",".","3",".",".","1"]
,["7",".",".",".","2",".",".",".","6"]
,[".","6",".",".",".",".","2","8","."]
,[".",".",".","4","1","9",".",".","5"]
,[".",".",".",".","8",".",".","7","9"]]
**Output:** true

**Example 2:**

**Input:** board = 
[["8","3",".",".","7",".",".",".","."]
,["6",".",".","1","9","5",".",".","."]
,[".","9","8",".",".",".",".","6","."]
,["8",".",".",".","6",".",".",".","3"]
,["4",".",".","8",".","3",".",".","1"]

,[".","6",".",".",".",".","2","8","."]
,[".",".",".","4","1","9",".",".","5"]
,[".",".",".",".","8",".",".","7","9"]]
**Output:** false
**Explanation:** Same as Example 1, except with the **5** in the top left corner being modified to **8**. Since there are two 8's in the top left 3x3 sub-box, it is invalid.

**Constraints:**

- `board.length == 9`
- `board[i].length == 9`
- `board[i][j]` is a digit `1-9` or `'.'`.

1. Using Hash Sets
Explanation
This approach leverages three separate lists of hash sets to keep track of the numbers seen in each row, column, and 3x3 box.

Initialization:

Create three lists, each containing 9 empty sets:
rows[9] for tracking numbers in each row.
columns[9] for tracking numbers in each column.
boxes[9] for tracking numbers in each 3x3 box.
Iteration:

Traverse each cell (i, j) in the 9x9 board.
If the cell contains a digit ('1' to '9'):
Determine the corresponding box index using (i // 3) * 3 + (j // 3).
Check if the digit already exists in rows[i], columns[j], or boxes[box_index].
If it does, the Sudoku is invalid (return False).
If not, add the digit to the respective sets.
Validation:

If no duplicates are found after checking all cells, the Sudoku board is valid (return True).
```Python
class Solution:
    def isValidSudoku(self, board: List[List[str]]) -> bool:
        rows = [set() for _ in range(9)]
        columns = [set() for _ in range(9)]
        boxes = [set() for _ in range(9)]
    
        for i in range(9):
            for j in range(9):
                num = board[i][j]
                if num == '.':
                    continue
                box_index = (i // 3) * 3 + (j // 3)
                if num in rows[i] or num in columns[j] or num in boxes[box_index]:
                    return False
                rows[i].add(num)
                columns[j].add(num)
                boxes[box_index].add(num)
        return True
```

Complexity Analysis
Time Complexity:
O(1)
The board size is fixed at 9x9, so the number of operations doesn't scale with input size.

Space Complexity:
O(1)
Fixed-size storage for rows, columns, and boxes sets.

2. Using Bit Manipulation
Explanation
This method optimizes space by using integers as bitmasks to track seen numbers instead of hash sets.

Initialization:

Create three lists of integers:
rows[9] for rows.
columns[9] for columns.
boxes[9] for 3x3 boxes.
Each integer will use its bits to represent the presence of digits 1-9.
Iteration:

Traverse each cell (i, j) in the board.
If the cell contains a digit:
Calculate the bitmask for the digit: 1 << (int(num) - 1).
Determine the box index with (i // 3) * 3 + (j // 3).
Use bitwise AND to check if the digit's bit is already set in rows[i], columns[j], or boxes[box_index].
If the bit is set in any, the Sudoku is invalid (return False).
Otherwise, set the bit in all three trackers using bitwise OR.
Validation:

If no duplicates are detected after checking all cells, return True.
```Python
class Solution:
    def isValidSudoku(self, board: List[List[str]]) -> bool:
        rows = [0] * 9
        columns = [0] * 9
        boxes = [0] * 9
    
        for i in range(9):
            for j in range(9):
                num = board[i][j]
                if num == '.':
                    continue
                mask = 1 << (int(num) - 1)
                box_index = (i // 3) * 3 + (j // 3)
                if (rows[i] & mask) or (columns[j] & mask) or (boxes[box_index] & mask):
                    return False
                rows[i] |= mask
                columns[j] |= mask
                boxes[box_index] |= mask
        return True
```

Complexity Analysis
Time Complexity:
O(1)
Fixed 9x9 board ensures constant time operations.

Space Complexity:
O(1)
Uses fixed-size integers for tracking.

3. Single Pass with Combined Checks
Explanation
This approach uses a single set to store all seen numbers along with their corresponding row, column, and box identifiers.

Initialization:

Create an empty seen set to store tuples representing the occurrence of a number in a row, column, or box.
Iteration:

Traverse each cell (i, j) in the board.
If the cell contains a digit:
Create three unique identifiers for the digit's presence in its row, column, and box:
(num, 'row', i)
(num, 'col', j)
(num, 'box', i//3, j//3)
Check if any of these identifiers already exist in the seen set.
If any exist, the Sudoku is invalid (return False).
If not, add all three identifiers to the seen set.
Validation:

If no duplicates are found after checking all cells, return True.
```Python
class Solution:
    def isValidSudoku(self, board: List[List[str]]) -> bool:
        seen = set()
        for i in range(9):
            for j in range(9):
                num = board[i][j]
                if num == '.':
                    continue
                if ((num, 'row', i) in seen or
                    (num, 'col', j) in seen or
                    (num, 'box', i//3, j//3) in seen):
                    return False
                seen.add((num, 'row', i))
                seen.add((num, 'col', j))
                seen.add((num, 'box', i//3, j//3))
        return True

```

Complexity Analysis
Time Complexity:
O(1)
The fixed size of the board ensures constant time operations.

Space Complexity:
O(1)
The seen set holds a maximum of 3 * 81 entries, which is constant.

# [[- Arrays and Hashing]]
# [347. Top K Frequent Elements](https://leetcode.com/problems/top-k-frequent-elements/)

#array #medium 
[[- Medium Level]]
Given an integer array `nums` and an integer `k`, return _the_ `k` _most frequent elements_. You may return the answer in **any order**.

**Example 1:**

**Input:** nums = [1,1,1,2,2,3], k = 2
**Output:** [1,2]

**Example 2:**

**Input:** nums = [1], k = 1
**Output:** [1]

**Constraints:**

- `1 <= nums.length <= 105`
- `-104 <= nums[i] <= 104`
- `k` is in the range `[1, the number of unique elements in the array]`.
- It is **guaranteed** that the answer is **unique**.


My Code:

```python
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        hashmap = defaultdict(list)
        for num in nums:
            hashmap[num] = hashmap.get(num, 0) + 1

        sort = {k: v for k, v in sorted(hashmap.items(), key=lambda item: item[1])}

        ans = []
        for i in range(1,k+1):
            ans.append(list(sort.keys())[-i])
        return ans
```

# Intuition

A Python dictionary can be used to store unique values as well as the frequency of their occurence. Once obtained, we return the top k most frequently occuring elements.

# Approach

- Count frequencies using a dictionary.
- Sort the elements by frequency in descending order.
- Return the first k elements from this sorted list.

# Complexity

- Time complexity: The costliest operation here is sorting, which operates in O(nlogn) time complexity. Note that this solution can be further improved to O(nlogk) complexity by using max-heaps to store the key of the most frequently occuring values.

- Space complexity: Creating an additional dictionary makes the space complexity O(n).

# Code

```java
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:

        orb = {}

        for i in nums:
            if i in orb:
                orb[i] += 1
            else: orb[i] = 1
        
        sorted_orb = sorted(orb, key=orb.get, reverse=True)
        return sorted_orb[:k]
                    
```


### The Bucket Sort Approach ($O(N)$ Time)
[[- Bucket Sort]]
The magic of this approach relies on the fact that the maximum possible frequency of any number is the length of the `nums` array itself. If `nums` has 6 elements, a number can appear at most 6 times.

Instead of sorting the frequencies, we use the _indexes_ of a new array to represent the frequencies.

```python
from typing import List
from collections import Counter

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        # 1. Count the frequencies (same as your dictionary map!)
        # Counter is just a shortcut for building that dictionary
        count = Counter(nums) 
        
        # 2. Create our "buckets"
        # We need an array of lists. The size is len(nums) + 1 so we can 
        # use the index to perfectly match the frequency (including 0).
        freq = [[] for _ in range(len(nums) + 1)]
        
        # 3. Populate the buckets
        # If the number '3' appears '2' times, we put '3' into freq[2]
        for num, frequency in count.items():
            freq[frequency].append(num)
            
        # 4. Gather the top k elements
        res = []
        # Loop backwards starting from the highest possible frequency
        for i in range(len(freq) - 1, 0, -1):
            for num in freq[i]: # Check if any numbers have this frequency
                res.append(num)
                if len(res) == k:
                    return res
```

### The Min-Heap Approach ($O(N \log k)$ Time)
[[- Heaps]]
If you haven't worked with heaps much yet, a **Min-Heap** is a special tree-like data structure that automatically keeps the smallest element at the very top. In Python, the `heapq` module makes this easy.

By pushing elements into the heap and immediately popping the top element whenever the heap gets larger than `k`, we effectively kick out all the low-frequency numbers, leaving only the `k` most frequent ones behind.


```python
import heapq
from typing import List
from collections import Counter

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        # 1. Count the frequencies
        count = Counter(nums)
        
        # 2. Build the heap
        heap = []
        for num, frequency in count.items():
            # We push a tuple. Python's heap sorts by the first item in the tuple, 
            # so we put frequency first!
            heapq.heappush(heap, (frequency, num))
            
            # If our heap grows larger than k, pop the smallest element out
            if len(heap) > k:
                heapq.heappop(heap)
                
        # 3. Extract the results
        res = []
        # The heap now contains exactly the k most frequent elements.
        for frequency, num in heap:
            res.append(num)
            
        return res
```


We will use the classic example:

`nums = [1, 1, 1, 2, 2, 3]`

`k = 2`

### Approach 1: Bucket Sort Walkthrough

**Step 1: Count the frequencies**

We iterate through `nums` and build our dictionary.

- **`count` dictionary:** `{1: 3, 2: 2, 3: 1}`
    

**Step 2: Create empty buckets**

We create an array of empty lists. The size is `len(nums) + 1` (which is 6 + 1 = 7). The index represents the frequency.

- **`freq` array:** `[ [], [], [], [], [], [], [] ]`
    
- _(Indices are: 0, 1, 2, 3, 4, 5, 6)_
    

**Step 3: Populate the buckets**

We look at each `(number, frequency)` pair in our `count` dictionary and place the `number` into the `freq` array at the index matching its `frequency`.

- Look at `1`: It has a frequency of 3. We append `1` to `freq[3]`.
    
    - **`freq` array:** `[ [], [], [], [1], [], [], [] ]`
        
- Look at `2`: It has a frequency of 2. We append `2` to `freq[2]`.
    
    - **`freq` array:** `[ [], [], [2], [1], [], [], [] ]`
        
- Look at `3`: It has a frequency of 1. We append `3` to `freq[1]`.
    
    - **`freq` array:** `[ [], [3], [2], [1], [], [], [] ]`
        

**Step 4: Gather the top k elements**

We iterate backward through the `freq` array, starting from index 6 down to 1. We have an empty result array: `res = []`.

- **Index 6:** `[]` -> Do nothing.
    
- **Index 5:** `[]` -> Do nothing.
    
- **Index 4:** `[]` -> Do nothing.
    
- **Index 3:** `[1]` -> Append `1` to `res`.
    
    - **`res` array:** `[1]`
        
    - Is `len(res) == k`? (1 == 2) -> False.
        
- **Index 2:** `[2]` -> Append `2` to `res`.
    
    - **`res` array:** `[1, 2]`
        
    - Is `len(res) == k`? (2 == 2) -> True! **Return `res`.**

### Approach 2: Min-Heap Walkthrough

**Step 1: Count the frequencies**

Just like before, we build our dictionary.

- **`count` dictionary:** `{1: 3, 2: 2, 3: 1}`
    

**Step 2: Build the heap**

We create an empty list to act as our heap: `heap = []`.

We iterate through our dictionary items: `(1, 3)`, `(2, 2)`, `(3, 1)`. Remember, we push them into the heap as `(frequency, number)`.

- **Item 1: number 1, frequency 3**
    
    - Action: Push `(3, 1)` into the heap.
        
    - **`heap` state:** `[(3, 1)]`
        
    - Size is 1. (Not greater than `k`, do nothing).
        
- **Item 2: number 2, frequency 2**
    
    - Action: Push `(2, 2)` into the heap. Because it's a Min-Heap, Python automatically places the item with the smallest frequency at index 0.
        
    - **`heap` state:** `[(2, 2), (3, 1)]`
        
    - Size is 2. (Not greater than `k`, do nothing).
        
- **Item 3: number 3, frequency 1**
    
    - Action: Push `(1, 3)` into the heap. It has the smallest frequency, so it moves to the top.
        
    - **`heap` state:** `[(1, 3), (3, 1), (2, 2)]`
        
    - Size is 3. This is greater than `k` (2).
        
    - Action: Pop the smallest element out. The heap removes `(1, 3)`.
        
    - **`heap` state:** `[(2, 2), (3, 1)]`
        

**Step 3: Extract the results**

Our heap now strictly contains the `k` items with the highest frequencies. We iterate through the remaining tuples in the heap to extract just the numbers.

- Empty result array: `res = []`
    
- Look at `(2, 2)`: Append number `2`. -> `res = [2]`
    
- Look at `(3, 1)`: Append number `1`. -> `res = [2, 1]`
    

**Return `res`.** _(Note: For this LeetCode problem, returning `[2, 1]` is just as valid as returning `[1, 2]`, as the order of the final output does not matter)._

You are climbing a staircase. It takes `n` steps to reach the top.

Each time you can either climb `1` or `2` steps. In how many distinct ways can you climb to the top?

**Example 1:**

**Input:** n = 2
**Output:** 2
**Explanation:** There are two ways to climb to the top.
1. 1 step + 1 step
2. 2 steps

**Example 2:**

**Input:** n = 3
**Output:** 3
**Explanation:** There are three ways to climb to the top.
1. 1 step + 1 step + 1 step
2. 1 step + 2 steps
3. 2 steps + 1 step

**Constraints:**

- `1 <= n <= 45`

### Problem Statement

You are climbing a staircase with `n` steps. At each step, you can either climb 1 or 2 steps. Your task is to determine the number of distinct ways you can climb to the top.

### Python Implementation

Here is the Python code that solves the problem efficiently:


```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 3: return n

        prev1 = 3
        prev2 = 2
        cur = 0

        for _ in range(3, n):
            cur = prev1 + prev2
            prev2 = prev1
            prev1 = cur
        
        return cur
```

### Step-by-Step Explanation

#### 1. **Base Cases**

The function handles small values of `n` directly:

- If `n = 1`, there is only one way to climb the stairs: a single step.
- If `n = 2`, there are two ways: (1 + 1) or (2).
- If `n = 3`, there are three ways: (1 + 1 + 1), (1 + 2), or (2 + 1).

For these cases, the function returns `n` immediately.

#### 2. **Dynamic Programming Initialization**

To compute the number of ways for larger values of `n`, we use three variables:

- `prev1`: Stores the number of ways to climb to the previous step (initialized to 3).
- `prev2`: Stores the number of ways to climb to the step before that (initialized to 2).
- `cur`: Tracks the current number of ways to climb the current step (initialized to 0).

#### 3. **Iterative Calculation**

Starting from step 4 (index 3 in 0-based indexing), the algorithm calculates the number of ways to climb to the current step using the formula:

```
cur = prev1 + prev2
```

This formula is derived from the fact that to reach the current step, you can either:

- Take a single step from the previous step (`prev1`), or
- Take a double step from the step before the previous step (`prev2`).

#### 4. **Update State Variables**

After computing `cur`, the variables `prev2` and `prev1` are updated to move the window forward:

- `prev2 = prev1`
- `prev1 = cur`

This ensures that the algorithm only keeps track of the necessary state for the current and previous two steps, optimizing memory usage.

#### 5. **Return the Result**

Once the loop finishes, `cur` contains the number of ways to climb to the top of the staircase.

### Example Walkthrough

Consider the input `n = 5`. Here’s how the variables evolve:

|Step Index|`prev2`|`prev1`|`cur`|
|---|---|---|---|
|3|2|3|5|
|4|3|5|8|
|5|5|8|13|

The result is `13`, indicating there are 13 distinct ways to climb 5 steps.

### Complexity Analysis

- **Time Complexity**: The algorithm processes each step once, resulting in `O(n)` time complexity.
- **Space Complexity**: Only three variables (`prev1`, `prev2`, and `cur`) are used, so the space complexity is `O(1)`.

### Why This Approach Works

This solution follows the principles of dynamic programming by breaking the problem into smaller overlapping subproblems. By using only three variables, it optimizes memory usage compared to a full array-based implementation.

### Conclusion

The "Climbing Stairs" problem is a simple yet powerful example of dynamic programming in action. By understanding its Fibonacci-like structure and implementing an iterative solution, we achieve both clarity and efficiency.
